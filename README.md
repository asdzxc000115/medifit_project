## 📖 프로젝트 개요 (Overview)
**MediFit**은 디지털 기기에 익숙하지 않은 **5060 시니어 세대**를 위해 설계된 직관적인 건강 관리 애플리케이션입니다. 복잡한 병원 예약, 흩어진 진료 기록, 기억하기 힘든 복약 관리를 **AI 챗봇**과 **단순한 UI**로 해결합니다.

| 항목 | 내용 |
| :--- | :--- |
| **진행 기간** | 2025.07 ~ 2025.09 (약 2개월) |
| **참여 대회** | 은하톤 8팀 프로젝트 |
| **팀 구성** | 총 6명 (리더 1, 팀원 5) |
| **나의 역할** | **Team Leader, Backend Developer** |

### 🛠️ 기획 배경
* **문제점(Pain Point):** 시니어 층은 모바일 앱의 복잡한 UI/UX로 인해 디지털 헬스케어 서비스 접근에 어려움을 겪습니다.
* **해결책(Solution):** 큰 글씨와 직관적인 UI, 그리고 자연어 처리가 가능한 AI 챗봇을 통해 "말하듯이" 건강을 관리할 수 있는 서비스를 개발했습니다.

## 👨‍💻 담당 직무 및 성과 (My Role)

**Role: Project Leader & Lead Backend Developer**

#### 1. 백엔드 아키텍처 설계 및 API 개발
* **RESTful API 구축:** 회원가입(JWT), 병원 예약, 진료 기록 조회 등 핵심 비즈니스 로직을 Spring Boot로 구현했습니다.
* **보안 강화:** Spring Security와 JWT를 도입하여 사용자 인증/인가 프로세스를 안전하게 처리했습니다.

#### 2. 데이터베이스 모델링 (ERD)
* **스키마 설계:** `User`, `Hospital`, `Reservation`, `Medicine`, `MedicalRecord` 등 주요 엔티티 간의 관계(1:N, N:M)를 정규화하여 설계했습니다.
* **데이터 무결성:** MySQL을 활용하여 데이터의 일관성을 보장하고 효율적인 쿼리를 작성했습니다.

#### 3. AI 챗봇 기능 구현
* **프롬프트 엔지니어링:** 시니어 사용자가 이해하기 쉬운 친절한 어투로 응답하도록 AI 모델의 시스템 프롬프트를 최적화했습니다.
* **API 연동:** 사용자의 자연어 입력을 백엔드 서버를 거쳐 AI 모델로 전달하고 응답받는 파이프라인을 구축했습니다.

## 📉 트러블 슈팅 및 회고 (Retrospective)

### 🚧 Challenges (시행착오)
1.  **일정 관리의 병목:** 초기 기획 단계가 길어지면서 실제 개발 기간이 촉박해지는 문제가 발생했습니다.
2.  **팀워크 및 역량 차이:** 1학년 팀원들이 다수 포함되어 있어 기술적 난이도 조절과 업무 분배에 어려움이 있었습니다.
3.  **리더의 과부하:** 백엔드 개발과 PM 역할을 동시에 수행하다 보니 특정 시기에 업무가 과중되었습니다.

### ✅ Solutions & Learned (해결 및 배운 점)
1.  **Agile한 우선순위 선정:** 필수 기능(MVP)과 부가 기능을 명확히 구분하고, 핵심 기능 구현에 집중하여 마감 기한을 준수했습니다.
2.  **지식 공유와 멘토링:** 팀원들에게 필요한 기술 가이드라인을 문서화하여 제공하고, 코드 리뷰를 통해 함께 성장하는 문화를 만들었습니다.
3.  **'완성'의 가치:** 완벽함보다는 **정해진 기간 내에 동작하는 결과물**을 만들어내는 것이 협업 프로젝트에서 가장 중요하다는 것을 깨달았습니다.
